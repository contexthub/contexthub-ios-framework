Overview

The CCHVault should be used to persist data to the ContextHub Vault API. This class provides methods for creating data in containers. Once you persist data, you can retrieve it by Container and Id.

Note: You can think of a container as a bucket for common data.

Instance Methods

createItem:container:attachments:completionHandler:

Creates items in the ContextHub Vault.

- (void)createItem:(NSDictionary *)item container:(NSString *)containerName attachments:(NSArray *)attachments completionHandler:(vaultCompletionBlock)completionHandler

Parameters

item

the NSDictionary representation of the item you want to persist.

containerName

the name of the container that will be used to store similar data.

attachments

(optional) an array of CCHVaultResource objects.

completionHandler

(optional) called when the request completes. The block is passed an NSDictionary representation of the item. If an error occurs, the NSError will be passed to the block.

Discussion

Note: the dictionary that is returned will have a key name vault_info. This object contains the id and other metadata that is used to work with the item on the ContextHub server.

Note: the name used in the CCHVaultResource will be added as a property on the item. The property will contain the URL that you can use to download the file.
